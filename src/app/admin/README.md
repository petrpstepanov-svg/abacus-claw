# Админ-панель AutoHub Rostov

## Обзор

Панель управления заявками с аутентификацией, фильтрами и повторной отправкой уведомлений.

## Маршруты

| Маршрут | Описание |
|---------|----------|
| `/admin` | Страница логина |
| `/admin/leads` | Таблица заявок с фильтрами и пагинацией |

## API

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/api/admin/leads` | Список заявок (фильтры, пагинация) |
| PATCH | `/api/admin/leads/[id]` | Переотправка уведомлений |

## Аутентификация

1. Установите `ADMIN_PASSWORD` в `.env`
2. Откройте `/admin` и введите пароль
3. Пароль сохраняется в `localStorage`
4. API защищён через `Authorization: Bearer <password>`
5. Middleware проверяет все запросы к `/api/admin/*`

## Фильтры

- **Тип заявки**: Все / Эвакуатор / Автосервис / Комиссар / Агрегаты
- **Период**: Все время / Сегодня / Неделя / Месяц
- **Уведомления**: Все / Отправлены / Не отправлены

## Компоненты

- `StatusBadge` — статус обработки (зелёный/жёлтый)
- `LeadTypeBadge` — тип заявки (цветовой бейдж)
- `LeadsTable` — таблица с данными заявок
- `LeadsFilters` — блок фильтров
- `Pagination` — навигация по страницам
